볼트 task 노트의 진행 상황을 `oobs note` 셸 명령으로 기록한다.
진행 일지 append, frontmatter 현황 요약(note) 갱신, 상태 변경, 노션 미러까지 전부 CLI 가 처리한다.

## 흐름

1. 슬래시 줄에서 **업무 검색어** 추출 (반드시 명시 — tasks/ 파일명·본문 매칭)
2. 메모 텍스트 결정:
   - **사용자가 메모를 명시했으면 그대로 사용 (LLM 가공 X)** — 보안 우선
   - 명시 안 됐으면 세션 컨텍스트의 최근 작업·결정을 **한 줄로** 요약 (60자 권장, 100자 한계)
3. 메모가 상태 변화를 의미하면 `--status` 추가 (예: "배포함" → 모니터링 또는 완료, "검토 요청" → 의사결정 대기).
   완료/취소는 `oobs done "<검색어>" "<메모>"` (또는 `--cancel`) 사용 — completed 기록과 데일리 로그까지 처리됨
4. 미리보기 출력 (실행할 명령) + 확인 (`OK` / `수정` / `취소`)
5. OK 면 실행:
   ```bash
   oobs note "<검색어>" "<메모>" [--status "<상태>"]
   ```
   - 다중 매칭이면 CLI 가 번호 선택을 요구하며 비인터랙티브에선 실패함 → 검색어를 좁혀 재시도하거나, 매칭 후보를 사용자에게 보여주고 고르게 한 뒤 `--first` 가 아닌 좁힌 검색어로 실행
   - `[미러 경고]` 출력이 보이면 사용자에게 알릴 것

## 사용 예

```
/oobs-note mdn 가드 기획 검토 통과, 내일 운영 배포
/oobs-note 정렬 버그          ← 메모는 세션 컨텍스트에서 한 줄 요약
```

## 안전 가드

- 민감 정보 (토큰·비밀번호·고객 실명) 는 일반화하거나 제외
- 민감한 진행 상황이면 셸에서 직접: `oobs note "검색어" "메모"` (Claude 가공 X)
