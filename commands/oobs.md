description: 옵시디언 볼트에 작업 노트 생성 — 세션 컨텍스트 자동 요약 (oobs CLI 래퍼, 병행 기간 노션 미러)
---

사용자가 `/oobs` 를 호출하면, 지금까지의 세션 대화·작업 내용을 바탕으로 `oobs new` 셸 명령으로 `~/vault/tasks/` 에 작업 노트를 만든다.
파일 쓰기·데일리 로그·노션 미러는 전부 CLI 가 처리한다 — Claude 는 맥락 추출과 플래그 구성만 담당.
볼트 스키마 참고: `~/vault/CLAUDE.md`.

## 흐름

1. 세션 컨텍스트에서 추출:
   - `--title`: 한 줄 요약. Jira 키가 있으면 `WL-1234 제목` 형태
   - `--status`: 시작 전 | 진행 중 | 배포 대기 | 모니터링 | 의사결정 대기 | 보류 | 취소 | 완료 (기본: 진행 중)
   - `--category`: 리팩토링 | 기존기능개선 | 신규기능개발 | 운영 — 맞는 게 없으면 자유 값 가능 (노션 미러에서만 자동 생략됨)
   - `--repo` / `--priority`(high|normal|low) / `--project`(글로벌 윙크|한국 윙크, 노션 미러용)
   - `--link`: Jira 티켓 등 대표 URL 1개 (같은 세션에서 `/tako` 호출이 있었으면 그 URL)
   - `--note`: 현황 요약 한 줄 (60자 권장)
   - `--due` / `--scheduled`: 자연어 ("금요일까지") → `YYYY-MM-DD`
   - `--session`: 현재 세션 이름/ID
   - `--body`: 배경/맥락 마크다운. **`## 진행 일지` 섹션은 넣지 말 것** — CLI 가 자동 생성
2. 미리보기 출력 (실행할 `oobs new ...` 명령 전체) + 확인 받기 (`OK` / `수정` / `취소`)
3. OK 면 실행. 출력의 `✓ 노션 미러` 줄로 미러 성공 여부 확인 — `[미러 경고]` 가 보이면 사용자에게 알릴 것

## 안전 장치

- 제목·본문은 LLM 초안 — 미리보기 단계에서 사용자가 수정 가능
- 토큰·비밀번호·고객 실명 같은 민감 정보는 본문에서 의식적으로 제외
- 동명 노트가 있으면 CLI 가 거부함 — 갱신은 `/oobs-note` 로 안내

## 사용 예

```
/oobs 방금 작업한 정렬 버그 수정 — 금요일까지 배포
/oobs                ← 세션 전체 맥락으로 자동 구성
```

## 노션 미러

미러 ON/OFF 는 `~/.config/oobs/config.yaml` 의 `mirror:` 가 단일 스위치 (기본 ON).
옵시디언 완전 이전이 결정되면 그 값만 `false` 로 바꾸면 된다.
